As I began the conceptualization of my Sinatra project, I really wanted nothing more than a simple app that would fulfill the requirements of the project.  I soon realized that I could build something useful, and though I was forced to change gears to make the app fit the mold, I was able to build something that would be useful and scalable.

I ran into a couple little hiccups, as I usually do when creating a project, as I didn't want to create an app that directly mirrors anything I've built before.  This was difficult to achieve, as certain aspects of the project needed to be built using pretty close to the same code I've used before.  Eventually, I said screw it, I need to build on what I know. 

I knew that certain aspects of my code might be a bit more complex, and I planned accordingly.  I started with the classes and methods I knew would not need much tweaking, then moved on to the code I knew was going to be a bit more creative.

In the end I was able to build something that I feel can be useful, and I hope others can find it useful as well.
